Book Repository

Overview

A system to manage a collection of books in. The system represents individual books with attributes such as title, author, ISBN, edition, publish year, and number of copies.

The system provides various functionalities including loading book data from a file, adding new books, borrowing books, returning books, removing books, and printing the details of available books.

This assignment emphasizes the application of object-oriented programming principles, file input/output operations, and collection frameworks.

Book Repository Operations

  • Load Book Data: This operation reads data from a CSV file, creates book objects from the data, and stores them in an arryList.
  • Add New Book: This operation prompts the user to input details of a new book (such as title, author, ISBN, edition, publish year, and number of copies) and adds the new book to the list.
  • Borrow Book: This operation allows a user to borrow a book by specifying its title. If the book is available (i.e., it has at least one copy), the number of copies is decremented by one.
  • Return Book: This operation allows a user to return a borrowed book by specifying its title. The number of copies of the book is incremented by one.
  • Remove Book: This operation allows a user to remove a book from the list by specifying its title. If the book is found, it is removed from the list.
  • Print Available Books: This operation prints the details (title, author, ISBN, edition, publish year, and number of copies) of all books currently available in the list.

Implementation Requirements

  • A class named Book should be created to represent the Book Repository System.
  • Implement all instance fields and methods required to manage the book system.

Important Notes

  • Reading CSV File Data:
    • Data is structured in lines, where each line represents a record. Fields within a line are separated by commas. These fields are used to initialize the instance fields in the Book class when creating book objects.
    • Use a loop to read the file line-by-line. Split each line by commas to extract individual fields. Module 3 - File IO lecture provides detailed steps on reading files line-by-line and handling data.
  • Creating Book Objects from File Data:
    • For each line, split the data into respective fields.
    • Create a Book object using the extracted data.
    • Populate the Book object's attributes with the field values.
  • Using Getter and Setter Methods:
    • Getter Methods: Retrieve instance field values.
    • Setter Methods: Set or update instance field values.
